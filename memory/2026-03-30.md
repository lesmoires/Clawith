## 🏛️ GOVERNANCE MODEL — Two-Layer Oversight

### Architecture
```
┌─────────────────────────────────────────────────────────────┐
│              CLAWITH MAINTAINER (Oversight Layer)           │
│  - Can query Hetzner info (read-only by default)            │
│  - Can execute destructive ops (logged for audit)           │
│  - Audits Hetzner activity                                  │
│  - "Break glass" access when needed                         │
└─────────────────────────────────────────────────────────────┘
                           ↓ delegates / reports to
┌─────────────────────────────────────────────────────────────┐
│              DEVOPS MOIRIA (Operations Layer)               │
│  - Daily Hetzner operations                                 │
│  - Monitoring, alerts, routine tasks                        │
│  - Destructive ops: REQUIRES Guillaume approval             │
│  - First responder for infrastructure issues                │
└─────────────────────────────────────────────────────────────┘
```

### Critical Constraint
**DevOps Moiria runs ON moiria-claw** — the same server it manages.

**Implications:**
- Destructive ops on moiria-claw = self-destruction risk
- Special protection: `moiria-claw` in `PROTECTED_SERVERS`
- Guillaume approval MANDATORY for any destructive op on protected servers

### Governance Rules (Enforced in Backend)

**Destructive Tools (DevOps Moiria blocked without approval):**
```python
HETZNER_DESTRUCTIVE_TOOLS = {
    'hetzner_delete_server',
    'hetzner_rebuild_server',
    'hetzner_resize_server',
    'hetzner_power_off',      # Can destabilize services
    'hetzner_shutdown',       # Can destabilize services
    'hetzner_reboot',         # Can cause temporary downtime
}
```

**Protected Servers (Guillaume approval MANDATORY):**
```python
PROTECTED_SERVERS = {
    'moiria-claw': 'DevOps Moiria host — destruction = self-destruction.',
    'moiria-coolify': 'Coolify host — manages all deployments.',
}
```

**Error Message (when blocked):**
```
⚠️ **Destructive Operation Blocked**

Tool `hetzner_reboot` requires **Guillaume approval**.

**Action Required:** Guillaume must explicitly approve this operation. 
Please request intervention from @Guillaume or @Clawith Maintainer.

*Rationale: Destructive Hetzner operations can cause service disruption.*
```
