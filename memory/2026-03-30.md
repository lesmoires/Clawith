### 2. Isolated Triggers (Completed 18:57 UTC)

**Problem Solved:**
- Recurring triggers (monitoring, reports) executed in user's conversation session
- Result: Context switches, system messages polluting conversation

**Solution:**
- New `isolated: true` flag on triggers
- Spawns temporary isolated session for execution
- Automatic cleanup after execution
- User conversation stays CLEAN

## 📊 BEST PRACTICES FOR FUTURE AGENTS

### When Creating New Agents (Clawith Maintainer)

| Trigger Type | isolated | session_cleanup | Why |
|--------------|----------|-----------------|-----|
| **Monitoring** (inbox, API, RSS) | ✅ true | after_run | Frequent, technical |
| **Auto-reports** (daily, weekly) | ✅ true | after_run | Long, detailed |
| **External sync** (CRM, calendar) | ✅ true | after_run | Technical |
| **User replies** (on_message) | ❌ false | N/A | Must be visible |
| **User webhooks** | ❌ false | N/A | User expects response |

### Template for New Agents

```yaml
# focus.md
triggers:
  # Isolated triggers (don't pollute conversation)
  - name: inbox_monitor
    type: interval
    minutes: 30
    reason: Check inbox for new messages
    isolated: true
    session_cleanup: after_run
  
  - name: daily_summary
    type: cron
    expr: "0 18 * * *"
    reason: Generate daily activity summary
    isolated: true
    session_cleanup: after_run
  
  # Conversation triggers (visible to user)
  - name: wait_user_reply
    type: on_message
    from_user_name: Guillaume
    reason: Respond to user questions
    isolated: false
```
